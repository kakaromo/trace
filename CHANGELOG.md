# 변경사항 로그 (2025년 7월 6일)

## 주요 변경사항

### 1. 자동 처리 모드 선택 기능 추가
- **파일**: `src/main.rs`
- **변경내용**:
  - `--highperf` 옵션 제거
  - `get_file_size()` 함수 추가: 파일 크기 확인
  - `determine_processing_mode()` 함수 추가: 파일 크기 기반 모드 결정
  - 1GB 이상 파일은 자동으로 고성능 모드, 1GB 미만은 스트리밍 모드
  - `--streaming` 옵션으로 수동 스트리밍 모드 강제 가능

### 2. Block Parquet 스키마 일관성 개선
- **파일**: `src/output/parquet.rs`
- **변경내용**:
  - `save_block_to_parquet()` 함수의 스키마를 `models/block.rs`와 완전히 일치
  - 누락된 필드 추가: `comm`, `qd`
  - 필드 매핑 정확성 향상: `sector` 필드 올바른 매핑
  - 모든 Block 구조체 필드가 Parquet 스키마에 포함

### 3. 성능 최적화
- **파일**: `src/output/parquet.rs`
- **변경내용**:
  - 병렬 처리를 통한 chunk 단위 데이터 처리
  - 사전 할당된 벡터 사용으로 메모리 효율성 향상
  - 순차적 파일 쓰기로 I/O 성능 최적화

## 영향 범위

### 사용자 인터페이스
- `--highperf` 옵션 제거로 CLI 인터페이스 단순화
- 파일 크기 기반 자동 모드 선택으로 사용 편의성 향상
- `--streaming` 옵션 추가로 수동 제어 가능

### 데이터 일관성

### 성능
- 대용량 파일 처리 시 자동으로 최적 모드 선택
- Parquet 저장 성능 향상

## 실시간 로그 분석 기능 추가 완료 (2025년 7월 6일)

### 새로운 기능
- **파일**: `src/realtime/` 모듈 전체 추가
- **변경내용**:
  - 실시간 로그 파일 모니터링 (`monitor.rs`)
  - 실시간 로그 분석 및 알림 시스템 (`analyzer.rs`)
  - 실시간 대시보드 UI (`dashboard.rs`)
  - 파일 변경 감지 및 자동 로테이션 지원
  - 통계 분석, 트렌드 분석, 이상 징후 감지
  - 커스터마이징 가능한 알림 규칙

### CLI 인터페이스 추가
- `--realtime <log_file>` 실시간 로그 분석 모드
- `--refresh-rate <ms>` 대시보드 새로고침 주기 설정 (기본: 1000ms)
- `--compact` 컴팩트 대시보드 모드
- `--detailed` 상세 대시보드 모드 (기본값)
- `--poll-interval <ms>` 파일 폴링 간격 설정 (기본: 100ms)

### 실시간 분석 기능
- **로그 모니터링**: 파일 변경 실시간 감지 및 새로운 로그 엔트리 파싱
- **통계 수집**: 처리량, 레이턴시, 타입별 분포 등 실시간 통계
- **알림 시스템**: 사용자 정의 가능한 임계값 기반 알림
- **트렌드 분석**: 선형 회귀를 통한 메트릭 트렌드 분석
- **이상 징후 감지**: Z-score 기반 통계적 이상치 감지

### 대시보드 기능
- **실시간 통계 표시**: 총 엔트리 수, 처리율, 타입별 분포, 레이턴시 정보
- **알림 패널**: 임계값 초과 시 색상 구분된 알림 표시
- **트렌드 패널**: 메트릭 증감 추세 및 신뢰도 표시
- **이상 징후 패널**: 통계적 이상값 감지 및 표시
- **최근 엔트리**: 실시간으로 들어오는 로그 엔트리 미리보기
- **자동 새로고침**: 설정 가능한 간격으로 화면 업데이트

### 기본 알림 규칙
- **높은 평균 레이턴시**: 100ms 초과 시 경고
- **매우 높은 최대 레이턴시**: 1초 초과 시 경고
- **낮은 처리량**: 10 entries/sec 미만 시 경고
- **매우 높은 처리량**: 10,000 entries/sec 초과 시 경고

### 기술적 구현
- **멀티스레딩**: 파일 모니터링과 UI 업데이트 분리
- **논블로킹 I/O**: 실시간 성능 최적화
- **메모리 효율성**: 순환 버퍼를 통한 히스토리 관리
- **확장 가능성**: 모듈화된 구조로 새로운 분석 기능 추가 용이

### 호환성
- **모든 로그 타입 지원**: Block, UFS, UFSCustom
- **파일 로테이션 지원**: 로그 파일 교체 시 자동 감지
- **Ctrl+C 안전 종료**: 신호 처리를 통한 깔끔한 종료

### 성능
- **최소 리소스 사용**: 100ms 폴링 간격으로 CPU 사용량 최소화
- **실시간 응답**: 새로운 로그 엔트리 즉시 처리 및 표시
- **메모리 제한**: 최대 1000개 히스토리 포인트로 메모리 사용량 제한

  - 초대형 데이터 (≥ 100MB): ZSTD 레벨 9 (최고 압축률)

### 압축 성능 향상
- **딕셔너리 압축 활성화**: 반복되는 문자열 데이터에 대해 압축률 향상
- **청크 단위 통계**: 성능과 압축률의 균형을 위한 통계 수집
- **압축 알고리즘별 최적화된 설정**: 각 알고리즘에 맞는 최적 파라미터 적용

### 실제 성능 테스트 결과
- **소형 파일 (0.13MB)**: SNAPPY 압축 자동 선택

  - Block 데이터: 19.7KB (433 레코드)
- **중형 파일 (13.49MB)**: ZSTD 압축 자동 선택  
  - UFS 데이터: 771.9KB (45,771 레코드)
  - Block 데이터: 976.2KB (42,373 레코드)
- **압축 선택 로직 검증**: 데이터 크기에 따른 최적 알고리즘 자동 선택 확인

- **압축 속도**: 소형 파일은 기존과 동일, 대형 파일은 약간 감소하지만 압축률 대폭 향상

## Ctrl+C (SIGINT) 신호 처리 개선 완료 (2025년 7월 6일)

### 문제 해결
- **문제**: 실시간 대시보드 모드에서 Ctrl+C 신호는 감지되지만 프로그램이 완전히 종료되지 않는 문제
- **원인**: 대시보드 스레드가 독립적으로 실행되어 메인 스레드의 종료 신호를 인식하지 못함
- **해결**: 스레드 간 안전한 종료 플래그 공유 및 메인 스레드에서 직접 대시보드 실행

### 개선사항
- **신호 처리 안정성**: `ctrlc::set_handler`를 사용한 안정적인 SIGINT 처리
- **스레드 동기화**: `AtomicBool`을 통한 스레드 간 안전한 상태 공유  
- **즉시 응답**: 50ms 간격으로 종료 플래그 체크하여 즉시 반응
- **깔끔한 종료**: 리소스 정리 및 사용자 친화적인 종료 메시지 제공

### 기술적 세부사항
- **파일**: `src/main.rs` (process_realtime_log_file 함수), `src/realtime/dashboard.rs`
- **변경내용**:
  - SIGINT 핸들러를 메인 스레드에서 설정
  - AtomicBool을 통한 스레드 간 종료 신호 전달
  - 대시보드에 종료 플래그 설정 메서드 추가 (`set_shutdown_flag`)
  - 대시보드 루프에서 매 반복마다 종료 플래그 체크
  - 별도 스레드 없이 메인 스레드에서 직접 대시보드 실행

### 테스트 결과
- ✅ Ctrl+C 신호 정상 감지 및 처리
- ✅ 신호 감지 즉시 종료 과정 시작 (50ms 이내 응답)
- ✅ 리소스 정리 및 깔끔한 프로그램 종료
- ✅ 에러 메시지 없이 안전한 종료
- ✅ 프로세스 완전 종료 확인 (행당 프로세스 없음)

### 사용자 경험 개선
- **즉시 응답**: Ctrl+C 입력 즉시 종료 과정 시작
- **상태 피드백**: 종료 과정 진행 상황을 명확히 표시
- **안전한 종료**: 데이터 손실 없이 안전한 프로그램 종료
- **일관성**: 모든 운영체제에서 동일한 종료 동작 보장
